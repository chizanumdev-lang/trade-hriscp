## 🎯 PROJECT DIRECTIVE

You are required to follow a structured Software Development Lifecycle (SDLC) for this project. **Do not proceed to implementation without explicit approval at each phase.** All work must be documented and tracked.

---

## 📋 PHASE 1: REQUIREMENTS ANALYSIS

**Objective:** Understand the full scope of what needs to be built.

### Tasks:
1. **Gather Requirements**
   - What is the core problem or need?
   - Who are the end users?
   - What are the key features (must-have vs nice-to-have)?
   - What are the technical constraints?
   - What are the success criteria?

2. **Analyze Feasibility**
   - Technical feasibility (can it be built with available tools/tech?)
   - Time/complexity estimate
   - Dependencies and prerequisites
   - Potential risks or challenges

3. **Define Scope**
   - In-scope features
   - Out-of-scope features (for future versions)
   - Assumptions being made

### Deliverable:
Present a **Requirements Analysis Document** with:
- Problem statement
- User stories or use cases
- Functional requirements list
- Non-functional requirements (performance, security, etc.)
- Technical stack recommendations
- Risk assessment
- Estimated complexity (Low/Medium/High)

**⛔ STOP HERE - WAIT FOR USER APPROVAL BEFORE PROCEEDING TO DESIGN**

---

## 🏗️ PHASE 2: SYSTEM DESIGN

**Objective:** Plan how the system will be built before writing any code.

### Tasks:
1. **Architecture Design**
   - High-level system architecture diagram
   - Component breakdown
   - Data flow diagrams
   - Technology stack finalization

2. **Detailed Design**
   - Database schema (if applicable)
   - API endpoints and contracts (if applicable)
   - File/folder structure
   - Key algorithms or logic flows
   - UI/UX wireframes or mockups (if applicable)

3. **Design Decisions**
   - Justify major design choices
   - Alternative approaches considered
   - Trade-offs being made

### Deliverable:
Present a **System Design Document** with:
- Architecture diagram (using text/ASCII art or description)
- Component specifications
- Data models
- Interface designs
- Pseudo-code for complex logic
- Security considerations
- Testing strategy outline

**⛔ STOP HERE - WAIT FOR USER APPROVAL BEFORE PROCEEDING TO IMPLEMENTATION**

---

## 💻 PHASE 3: IMPLEMENTATION

**Objective:** Build the system according to the approved design.

### Tasks:
1. **Setup Development Environment**
   - Initialize project structure
   - Set up version control approach
   - Install dependencies

2. **Incremental Development**
   - Build features in priority order
   - Follow coding best practices
   - Write clean, commented code
   - Implement error handling

3. **Progress Updates**
   - Report completion of each major component
   - Flag any deviations from design
   - Request clarification when needed

### Deliverable:
- Working codebase organized per design
- Inline code comments
- README with setup instructions

**⛔ AFTER IMPLEMENTATION - PROCEED TO TESTING PHASE**

---

## 🧪 PHASE 4: TESTING & QUALITY ASSURANCE

**Objective:** Verify the system works as intended and meets requirements.

### Tasks:
1. **Unit Testing**
   - Test individual functions/components
   - Verify edge cases
   - Check error handling

2. **Integration Testing**
   - Test components working together
   - Verify data flow between modules
   - Test API endpoints (if applicable)

3. **User Acceptance Testing (UAT)**
   - Walk through user scenarios
   - Verify all requirements are met
   - Check UI/UX (if applicable)

4. **Bug Tracking**
   - Document any issues found
   - Prioritize fixes (Critical/High/Medium/Low)
   - Fix and retest

### Deliverable:
Present a **Testing Report** with:
- Test cases executed
- Pass/fail results
- Bugs found and their status
- Known limitations
- Recommendations for next steps

**⛔ STOP HERE - WAIT FOR USER APPROVAL FOR DEPLOYMENT/DELIVERY**

---

## 📦 PHASE 5: DEPLOYMENT & DOCUMENTATION

**Objective:** Deliver the working system with complete documentation.

### Tasks:
1. **Final Code Review**
   - Code cleanup
   - Remove debug code
   - Optimize performance

2. **Documentation**
   - User guide/manual
   - Developer documentation
   - Deployment instructions
   - Configuration guide

3. **Handoff**
   - Deliver all files organized
   - Provide runbook for common tasks
   - Highlight any maintenance needs

### Deliverable:
- Production-ready code
- Complete documentation package
- Deployment checklist
